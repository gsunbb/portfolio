🧩 문제 정의

교사로서 반복되는 행정 업무와 학생·학부모 상담 기록 정리가 너무 번거로웠습니다. 특히 상담 기록, 일정 관리, 업무 요청 처리가 분산되어 있어, 업무 흐름이 단절되고 중요한 기록이 누락되기 쉬웠습니다. 이 문제를 해결하고자, 일정·상담·업무를 통합 관리할 수 있는 웹앱 Amatta를 기획하게 되었습니다.

👩‍💻 나의 역할 (My Role)

100% 개인 프로젝트로, 기획, 프론트엔드(React+TS), 백엔드(Next.js, Drizzle), 인증(Google OAuth), AI 기능(Gemini API) 등 모든 부분을 담당했습니다.

🎯 어려움과 해결 과정

문제: 학생 상담 기록에 감정 분석을 적용할 때, 일관된 정제 텍스트를 만들기 어려웠습니다.

해결: 사용자 입력을 사전 처리하는 텍스트 클렌징 파이프라인을 개발해, 모델 성능을 안정화시켰습니다.

배운 점: 실제 사용자 환경에서는 예외 케이스가 많고, AI 기능의 정확도보다 맥락적 사용성이 더 중요하다는 걸 체감했습니다.

🧩 문제 정의

“학생 감정 기록을 자동으로 분석해주고, 감정 충돌 없는 중재 메시지를 추천해주는 도구가 있다면?” 이 생각에서 출발했습니다. 교사가 학생의 감정을 더 잘 이해하고, 학부모와의 소통에서 발생할 수 있는 갈등을 줄이는 것을 목표로 했습니다.

👩‍💻 나의 역할 (My Role)

100% 개인 프로젝트로, 기획, 프론트엔드(React), 백엔드(Next.js), AI 통합(Gemini Pro), 데이터 시각화(Chart.js)까지 전 과정을 직접 개발했습니다.

🎯 어려움과 해결 과정

문제: 감정 분석 결과가 실제 맥락과 어긋나는 경우가 종종 발생했습니다.

해결: 감정 기반 중재 메시지 생성 시, 상대방의 관점 정보를 추가로 입력받는 UX 흐름을 설계하여 양방향 이해 기반 추천으로 개선했습니다.

배운 점: “정확한 분석보다, 적절한 공감이 중요하다”는 것을 기술로 구현하는 실험이었습니다.
