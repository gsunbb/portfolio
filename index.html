100% 개인 프로젝트로, 교사의 요구사항 분석부터 기획, UI/UX 디자인, 프론트엔드 개발까지 전체 개발 과정을 담당했습니다. 특히 교실 환경에서 태블릿이나 노트북에서 쉽게 사용할 수 있는 반응형 UI를 중점적으로 디자인했습니다.

해결: 탭 기반 인터페이스와 직관적인 +/- 버튼 시스템을 도입했으며, 실시간으로 집계 상태를 보여주는 대시보드를 개발했습니다.

배운 점: 교육 현장의 특수성을 고려한 UX 설계가 중요하며, 기능성과 더불어 시각적 피드백이 사용자에게 주는 확신이 중요하다는 것을 배웠습니다.

🧪 Behind the Scene

나의 역할

어려움과 해결 과정
가장 큰 고민은 '얼마나 복잡하게 만들 것인가'였다. 너무 많은 기능을 넣으면 사용성이 떨어질 수 있고, 너무 단순하면 실제 사용에 제한이 생긴다. 결국 학교에서 가장 많이 사용하는 두 가지 선거 방식(남녀 구분/통합)을 중심으로 기능을 최적화했다.

가장 애정하는 디테일
남학생/여학생 후보자를 구분하는 카드 색상과 실시간으로 전체 득표 상황을 보여주는 대시보드. 특히 개표가 완료되면 상태 표시가 초록색으로 변하는 시각적 피드백이 마음에 든다. 작은 디테일이지만 사용자에게 큰 차이를 만드는 요소들이다.

🧩 문제 정의

학생들의 정서적 안정과 감정 관리는 학습 효과와 직결되지만, 교사가 매일 모든 학생의 감정 상태를 파악하고 기록하기는 어렵습니다. 특히 자신의 감정을 표현하기 어려워하는 학생들은 더욱 관심이 필요합니다. 이런 상황을 해결하기 위해 직관적이고 쉽게 감정 상태를 기록하고 통계화할 수 있는 시스템을 개발했습니다.

👩‍💻 나의 역할 (My Role)

100% 개인 프로젝트로, 사용자 경험 디자인부터 프론트엔드 개발, 데이터 시각화, 통계 분석 로직까지 모든 부분을 담당했습니다. 특히 교실에서 아이패드나 스마트보드에 적합한 인터페이스를 구현하는 데 중점을 두었습니다.

🎯 어려움과 해결 과정

문제: 감정을 표현하는 방식을 어떻게 디자인하고, 수집된 감정 데이터를 어떻게 의미 있게 시각화할 것인지가 주요 과제였습니다.

해결: 이모티콘 기반의 직관적인 카드 UI와 Chart.js를 활용한 시각화를 구현했으며, 수집된 데이터를 기반으로 주간 인사이트와 맞춤형 조언을 제공하는 로직을 개발했습니다.

배운 점: 감정 데이터는 단순한 수치가 아닌 패턴으로 봐야 하며, 시각화 방식에 따라 같은 데이터도 다르게 해석될 수 있다는 것을 배웠습니다.

🛠️ 기술 스택

Frontend: HTML5 CSS3 JavaScript (ES6+) Tailwind CSS
데이터 시각화: Chart.js
데이터 관리: Local Storage CSV Export
UI 요소: Font Awesome CSS Animation

🧪 Behind the Scene

문제 정의
아침 조회 시간에 학생들의 기분을 물어보는 게 일상이지만, 40명 가까운 학생들 모두와 소통하긴 어렵다. 게다가 점점 말수가 줄어드는 학생들이 많아지면서 감정 체크인이 꼭 필요하다고 느꼈다.

나의 역할
처음엔 단순히 카운팅만 되는 시스템을 구상했지만, 개발하면서 통계와 분석 기능의 중요성을 깨달았다. 특히 주간 감정 트렌드를 보여주는 기능이 가장 힘들었지만 가장 가치 있었다.

어려움과 해결 과정
감정은 주관적인데 어떻게 객관화할 것인가? 또한 교실에서 한 대의 태블릿으로 다수의 학생이 순차적으로 체크인하는 상황에서 어떻게 UI를 최적화할 것인가? 감정별 카드 인터페이스와 hover 시 나타나는 컨트롤러는 이런 고민의 결과물이다.

가장 애정하는 디테일
"오늘 기분이 좋지 않은 친구들이 많네요. 서로 위로해주는 하루가 되길 바라요." 같은 자동 생성 조언 메시지. 단순히 통계를 보여주는 것보다, 교사가 바로 학급 활동에 반영할 수 있는 인사이트를 제공하고자 했다. 작지만 의미 있는 디테일이다.
